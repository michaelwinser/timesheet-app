## 8. Implementation Order

1. **API Client** - Add rules methods to `client.ts` and types
2. **RuleCard** - Basic card component for list
3. **Rules Page** - List with empty state
4. **QueryInput** - Input with validation
5. **RuleEditor** - Create/edit form
6. **RulePreview** - Preview modal
7. **Navigation** - Add Rules link to AppShell
8. **Polish** - Loading states, error handling

## 9. Color Tokens

Use existing Tailwind classes:

- Enabled indicator: `bg-green-500`
- Disabled indicator: `bg-gray-300`
- Priority badge: `bg-yellow-100 text-yellow-800`
- Conflict warning: `bg-yellow-50 border-yellow-200`
- Error state: `bg-red-50 border-red-200 text-red-700`
